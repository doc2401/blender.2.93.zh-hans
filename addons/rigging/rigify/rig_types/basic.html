基础¶

这些装配器类型用于生成简单的单骨特征，并用于定制直接在元装配器中完成的装配器。

即使在连接的链条内，单骨装配器类型也必须单独应用于每块骨头，并且可以由不同的装配器类型控制连接的孩子。这与通常消耗整个连接链的基于链的装配器类型不同。

basic.copy_chain¶

复制骨骼链，使链中的所有父关系保持不变。可用作定制装配器的实用装配器类型。

要求：由至少两块连接的骨骼组成的链条。

布尔

启用后，将创建控制骨骼和小部件。

布尔

当启用时，将创建变形骨骼。

basic.pivot¶

单骨装配器类型，用于创建用于旋转和缩放其子子装配器的 "自定义轴心" 控件。

此类控件在旋转或缩放时转换其子控件，而移动控件仅更改旋转或缩放时使用的枢轴点。

主控制

启用后，将创建带有盒子小部件的额外父控件骨骼，以允许移动装配器。除 骨骼变形 外，所有其他选项也要求使用此选项。

父级

根据自定义属性的值生成用于切换装配器的有效父级的机制。

register

将装配器注册为其子子装配器的父开关的潜在父范围。

标签

为注册父作用域指定其他逗号分隔的标签关键字。它们可用于其他装配器筛选父级选项，或用于选择默认父级。

此处使用的一些现有标签：

注入 (特殊的)

父级作用域将提供给 父级 子装配器的所有子级，而不仅仅是此装配器的子级。

held_object

角色手中握持的对象的控件。由手指IK首选。

"" 注入， HOLD_OBJECT "的组合非常适合这种控件。

轴心控制

禁用此功能可避免生成实际的自定义数据透视控件，有效地将此装配器类型转换为具有父级切换支持和其他小部件的 basic.super_copy 版本。

变形骨骼

当启用时，将创建变形骨骼。

basic.raw_copy¶

复制骨骼而不带 "ORG-"名称前缀的单骨装配器类型。

通常，从元装配器复制的所有骨骼都前缀为 ORG- ，并放置在隐形层上。这排除了它们用作控制或变形的骨骼，这使得很难从元装配器传输复杂的完全自定义装配器逐字记录。

这种装配器类型不添加自动前缀，因此允许在元钻骨名称中手动包含适当的 "ORG-"、 "MCH-"或 "DEF-"前缀，或者不使用前缀来创建对照骨骼。

约束

允许重新定位属于骨骼的约束指向在生成装配器过程中创建的骨骼，从而允许自定义索具与生成的骨骼集成。

若要使用此功能，请在约束名称中添加 @``和预期目标骨骼名称，从而产生 ``... @ bone_name 语法。生成装配器的所有骨骼后，将替换约束目标骨骼。如果新骨骼名称仅为 CTRL 、 MCH 或 DEF ，则只会替换现有目标骨骼名称中的 ORG 前缀。对于骨架约束，您可以为每个目标添加 @ 后缀，或者仅添加一个 @CTRL 、@MCH 或 @DEF 后缀以更新所有内容。

父级

如果该字段不为空，则对骨骼的父级应用相同的名称替换逻辑。

启用此功能后，即使没有父级，骨骼也不会自动生成根骨；如果需要，请在 父级 字段中输入 "根" 。

basic.super_copy¶

简单地复制骨头的单骨装配器类型。可用作实用工具装配器类型，用于向装配器添加自定义功能或特定的骨骼变形。

布尔

启用后，将创建一个控制骨骼和小部件。

小工具（布尔）

When enabled a circle widget will be created in replacement to the standard.

布尔

当启用时，将创建变形骨骼。

约束

Works the same as in the basic.raw_copy rig.
