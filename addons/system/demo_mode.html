演示模式¶

待补充。

激活¶

  • 打开Blender并转到 "首选项" ，然后转到 "附加组件" 选项卡。

  • 单击系统，然后单击演示模式以启用脚本。

配置¶

这是从文件菜单中的 文件演示模式（设置） 完成-->的，您可以在此处选择递归搜索要循环的混合文件的目录。

此时，可以选择如何处理每个文件，主要选项是渲染或播放动画，其中包含播放多少个循环、渲染完成后等待的时间等选项。

通过发行版自带的安装包安装Blender，可以确保与系统中其他安装包的一致性，并且可以提供其它功能（包管理器提供），比如包列表，更新通知以及自动菜单配置。但是要注意的是，这些包可能不是官方最新版，或不包括Blender的某些功能。例如，有些发行版因为许可证的原因未能编译CUDA支持。¶

自动 *选项根据窗口布局使用图像/节点编辑器的情况，在 渲染 和 播放 选项之间进行选择。如果您愿意在所有文件上使用这些设置，则可以启用 立即运行*选项并确认。

修改配置¶

如果要为每个文件设置选项，您可以在设置期间禁用 立即运行 ，然后选择 "演示" 文本数据块。此文件每个文件有一行，可以使用混合文件编辑和保存设置。当 "演示" 存在时，您只需转到： menuselection ： "文件-->演示模式（开始）"。

属性¶

这些关键字参数显示在生成的`` demo::

dict(anim_cycles=2, anim_render=False, anim_screen_switch=0.0, anim_time_max=8.0, anim_time_min=4.0, display_render=4.0, file='foobar.blend', mode='AUTO'),

  • anim_cycles -- 动画播放次数。

  • anim_render --渲染整个动画（仅适用于 mode='RENDER' ）。

  • anim_screen_switch --切换屏幕之间的时间（以秒为单位）或0以禁用。

  • anim_time_max --显示动画的最大秒数（如果最终帧无缘无故非常高）。

  • anim_time_min --显示动画的最小秒数（用于小循环）。

  • display_render -- 在继续之前显示渲染图像的时间（以秒为单位）。

  • --log-file <filepath>

  • mode -- AUTO / PLAY / RENDER -- 加载时执行哪些动作.

便携化配置¶

设置 "演示" 后，您可能需要将其移至其他系统。在这种情况下，路径可能不匹配，如果匹配，请更新 search_path 变量以指向新系统上的路径。您还可以设置 search_path 以使用当前混合文件的目录:

search_path = "//"

用法¶

演示启动后，它将在同一Blender实例中播放、渲染和加载不同的文件。自使用模态运算符以来，您仍然可以在演示运行时与Blender一起使用。

如果要停止演示模式，可以按 Esc 并继续使用Blender。要再次启用，文件菜单右侧有三个按钮可导航 上一个 / 暂停 / 下一个 。

局限¶

  • 一个文件不能播放动画，然后渲染，您需要复制 演示 中的行，并将一个模式设置为 渲染 ，另一个模式设置为 播放。
